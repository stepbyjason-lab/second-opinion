# image-craft — 벤더무관 이미지 프롬프트 크래프트

이미지 생성 모델(gpt-image, Gemini 계열 등 무엇이든)에 공통으로 통하는 프롬프트 크래프트 원칙.
핵심: **모호한 형용사 대신 구체적 수치·명사로 채운다.** "예쁜 사진" 대신 "85mm, f/2.0, 측면
45도 소프트박스"처럼 사진가가 실제로 쓰는 어휘로 지시한다.

## 1. 피사체(subject) 묘사법

**유형**(인물/제품/사물/환경) → **외형**(나이대·특징적 디테일) → **의상/마감**(제품이면 소재·
마감·색상) → **표정/동작** → **프레임 내 위치**(중앙, 좌측 1/3 등) 순서로 명시하면 누락이 준다.
제품류는 외형 대신 소재·마감·색상·표면 텍스처를 구체화. 다중 피사체는 각각 위치를 명시해
겹침·혼동을 막는다.

## 2. 카메라 어휘

- **렌즈(mm)**: 16mm 광각(왜곡·환경) · 35mm 리포티지 · 50mm 표준 · 85mm 인물(무난) ·
  100mm 마크로(제품) · 135–200mm 망원(배경 압축)
- **조리개(f값)**: f/1.4–f/2.8 얕은 심도(감성·인물) · f/8–f/11 깊은 심도(제품·전체 선명)
- **앵글**: eye-level(중립) · low(위압·영웅) · high(취약·귀여움) · overhead(플랫레이) ·
  dutch(불안·역동) · three-quarter
- **프레이밍**: 클로즈업/미디엄/와이드 + "무엇이 프레임의 몇 %를 채우는가"까지
- **피사계심도**: shallow/medium/deep 을 조리개값과 중복 지정(오히려 안정적)
- **필름스톡(색감 단축키)**: Kodak Portra 400(따뜻한 피부톤) · Cinestill 800T(시네마·야간 네온) ·
  Fuji Pro 400H(쿨톤 파스텔) — 톤 조정을 말로 풀지 말고 필름명 하나로 대체

## 3. 조명 — 종류 + 방향 + 질감 세트로

- 종류: natural window / softbox / ring light / overhead / backlit / rim / golden hour / studio strobe
- 방향: "key 45도 카메라 왼쪽, fill 오른쪽 0.3 비율"처럼 각도+비율
- 질감: soft/diffused vs hard/specular, 그림자 경도

인물에서 링라이트는 인위적 캐치라이트를 남겨 "AI스러움"의 주범 — 자연광/소프트박스 45도가 안전 기본.

## 4. 스타일 / 무드 / 컬러그레이딩

aesthetic(전체 톤) + color grading(색 처리) + mood(감정 키워드) + post-processing(그레인·halation)을
분리해 각각 짧게. 자주 쓰는 방향: UGC 셀피 · 라이프스타일 · 스튜디오 제품 · 플랫레이 · 에디토리얼
헤드샷 · 시네마틱 · 미니멀 · 다큐멘터리 · 골든아워.

- **품질/해상도 서술어**: 프롬프트 끝에 "high resolution", "ultra detailed", "8K detail" 같은
  서술어를 덧붙이면 디테일이 올라간다.
- **스타일 모순 금지**: 상충하는 스타일을 동시에 지정하지 말 것(예: photorealistic + oil painting).

## 5. 구도

- 3분할 기본, 의도적으로 깰 때만 명시
- 여백(negative space)은 텍스트 오버레이 예정 위치를 미리 확보(좌측 1/3 또는 상·하단)
- 다중 요소는 전경/중경/배경 레이어를 각각 지정

## 6. 네거티브 프롬프트 (피할 것)

네거티브를 지원 안 하는 모델도 본문에 "avoid: ..."로 넣으면 유사 효과. 베이스:
```
blurry, low quality, distorted, extra fingers, watermark, cartoon, plastic skin, airbrushed, stock photo feel
```
상황별 추가 — 제품컷: `cluttered background, motion blur, hands in frame` · 뷰티: `heavy makeup,
plastic skin, beauty filter` · UGC(의도적 불완전): `professional studio, perfect lighting` ·
텍스트 포함: `gibberish text, distorted letters, melted typography`

> ⚠ 일부 모델은 네거티브 프롬프트를 아예 지원하지 않는다(예: gpt-image-2). 그 경우 "피할 것"을
> 나열하지 말고 **원하는 것을 긍정문으로** 묘사한다("no blur" 대신 "sharp focus"). §12 참조.

## 7. 플랫폼별 종횡비

| 비율 | 용도 | 비율 | 용도 |
|---|---|---|---|
| 1:1 | 인스타 피드·프로필·범용 | 16:9 | 유튜브 썸네일·웹 히어로 |
| 4:5 | 인스타 세로(화면 최대 점유) | 21:9 | 시네마틱 배너 |
| 9:16 | 스토리·Reels·Shorts | 3:2 / 2:3 | 인쇄·핀터레스트 |
| | | 3:4 / 4:3 | 매거진 내지·구형 표준 |

## 8. 텍스트 렌더링 팁 (실패 최다 영역)

- **권장: 요소당 5단어 이내**(길수록 뭉개짐; 정확한 단어수 한계는 벤더별 상이 §12) · **한 이미지 3~5개 요소 이하**(초과분은 후처리 오버레이)
- **색은 hex 코드로**(named color는 모델마다 해석 느슨) · 배치 구체적으로("하단좌측, 로고 아래")
- 한/중/일 문자는 "modern sans"·"elegant serif"·"handwritten brush" 범용 어휘로, 특정 폰트명 요구 금지
- 브랜드 로고는 렌더링 시도 말고 별도 합성

## 9. 피사체 일관성

- 시리즈(캐러셀·비포/애프터·멀티앵글)면 외형 핵심 블록(나이·특징·의상)을 모든 프롬프트에 **문자 그대로 반복**, 바뀌는 요소만 교체
- 레퍼런스 이미지 입력을 받는 모델이면 이전 생성물을 참조로 재투입이 텍스트 묘사보다 안정
- 시드 고정 지원 모델이면 시리즈 전체 동일 시드 — 단 거동은 벤더별 상이(§12)

## 10. 흔한 실패와 교정

| 증상 | 원인 | 교정 |
|---|---|---|
| 텍스트 깨짐 | 단어 과다·폰트 모호·named color | 5단어 이내, "modern sans", hex |
| 인물이 플라스틱 | 네거티브 부실·필름스톡 미지정 | `plastic skin, airbrushed, stock photo feel` + 필름스톡 |
| 배경 산만 | 배경 미지정 | 배경 요소 나열 or `cluttered background` 네거티브 |
| 안전필터 차단 | 민감하게 읽히는 묘사 | 모호성·민감 표현 제거 |
| 저작권 캐릭터/셀럽 거부 | 실명 요청 | 이름 대신 특징 묘사 |
| 결과가 밋밋·저품질 | 모호한 프롬프트("a nice picture") | 구체화 + "high resolution / ultra detailed" 추가 |

## 11. 좋은 프롬프트 체크리스트

- [ ] 피사체: 유형 + 외형 + 의상/마감 + 동작 + 위치
- [ ] 장소/시간/날씨
- [ ] 조명: 종류 + 방향 + 질감
- [ ] 카메라: 렌즈mm + 조리개 + 앵글 + 프레이밍 + 심도
- [ ] 스타일: 무드 + 컬러그레이딩 + (선택) 필름스톡 + 품질 서술어
- [ ] 네거티브(베이스 + 상황별) — 또는 미지원 모델이면 긍정문으로
- [ ] 텍스트 있으면: 5단어 이내 × 3~5개, hex 색, 배치
- [ ] 종횡비가 실제 사용 플랫폼과 일치

## 12. 벤더별 상이 가능 (단정 금지 — 플래그만)

원리는 유효하나 모델마다 지원·거동이 다를 수 있음:
- **네거티브 프롬프트 지원 여부** (gpt-image-2는 미지원 — 긍정문으로 대체)
- 시드 기반 일관성 지원 여부
- 구조화(JSON) 프롬프트의 정밀도 향상 효과 크기
- 네이티브 해상도 상한·K 선택 기준
- 레퍼런스 이미지 개수·마스크리스 편집 지원
- 텍스트 렌더링 단어수 한계(5단어는 실측 기준치)
- 안전필터 민감도
